# POR 投影看板 - 隱身雲端備份功能開發記錄 (2026-04-20)

本文件詳細紀錄了「台灣版/海外版」分流策略中的第一階段：**隱身雲端備份**功能的實施過程。

## 1. 變更內容說明

### 🟢 動態工作流管理 (Workflow Customization) [NEW]
- **階段編輯器**：在管理後台新增了「工作流階段定義」區塊，支援隨時新增、刪除、重新命名看板階段。
- **自適應佈局**：大螢幕看板 (`show.html`) 現在會根據設定的階段數量自動調整欄位寬度（Flexbox 驅動）。
- **同步機制**：修改階段後，設定會立即寫入 `config.inf` 並同步至看板畫面，無需修改代碼。

### 🟢 帳號授權限制 (License Control)
- **額度控管**：在 `config.inf` 新增 `max_users` 參數。
- **註冊攔截**：優化了「快捷新增用戶」邏輯。當一般帳號總數達到上限（預設 4 人）時，系統會攔截新增請求並提示「授權上限」。
- **超級後門**：保留 `administrator` 帳號作為緊急後援，不計入一般商務授權名額。

### 🟢 視覺與標題自定義 (Customization)
- **獨立設定頁面** [NEW]：看板全域設定（標題、色系、工作流）已遷移至獨立路由 `/dashboard_settings`。
- **主選單入口** [NEW]：在 `admin_home.html` 主選單中新增「看板設定」大按鈕，僅限管理員顯示，直覺管理全站配置。
- **介面分離**：實現了「資料操作」與「全域設定」的分離，管理進度頁面 (`manage.html`) 回歸純粹的資料轉移與管理功能。

### 🟢 隱身雲端備份
- **每日定時**：固定於凌晨 3:00 執行。
- **單檔保留**：Google Drive 永遠只留最新一份，節省空間且支援版本紀錄。
- **儲存按鈕**：強化為飽和度高的「深海藍」，按鈕文字更新為 **「儲存所有變更並授權用戶」**，更加符合專業管理語境。


### 🟢 實體路徑加固 (Data Persistence Hardening) [NEW]
- **遷移至 database/**：資料庫已從隱晦的實體資料夾移至專案根目錄的 `database/`。
- **絕對透明掛載**：更新了 `docker-compose.yml`，確保開發者在 D 槽能直觀看到資料檔案。
- **雙向同步驗證**：已通過內部測試，確認容器內的任何寫入都會即時反應在 Host 端檔案時間戳記上。

### 🟢 用戶管理修復 (Bug Fix)
- **表單分離重構** [NEW]：將「快捷新增」與「權限修改」拆分為兩個獨立表單。
    - **成果**：現在在新增用戶輸入框按 Enter，只會觸發新增操作，完全避開了之前的刪除/修改衝突。
    - **精確反饋**：新增用戶成功時僅顯示「建立成功」；修改表格權限時僅顯示「權限已保存」。
- **刪除功能修復**：修復了因之前的安全調整導致刪除功能失效的問題，現在刪除按鈕會正確觸發 POST 請求並保有二次確認彈窗。
- **通知分類系統**：成功時顯示綠色，額度不足或錯誤時顯示紅色警告。

## 2. 驗證結果
- **連線修復**：修復了路由重複導致的崩潰。
- **功能驗證**：標題、色系、工作流自定義均運作正常。
- **資料掌控**：透過 `database/` 資料夾實現了「物理級別」的監控。

## 3. 下一步操作 (User Action Required)
1.  **開始使用**：您現在可以開始輸入看板資料。
2.  **觀察變動**：您可以開著 D 槽視窗，每當您在後台按下「轉移」或「新增」時，應該能看到 `database/projection.sqlite` 的修改時間跳動。
3.  **備份設定**：若要啟用雲端備份，請記得在 `config.inf` 填入 `backup_folder_id`。
