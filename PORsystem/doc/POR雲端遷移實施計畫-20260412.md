# POR 系統遷移至 Cloudflare (方案 A) 實施計畫

本計畫旨在將現有的 Flask + SQLite 看板系統遷移至 Cloudflare 的邊緣運算平台，利用 **Cloudflare Workers (Python)** 作為後端邏輯，並使用 **Cloudflare D1** 作為分散式資料庫，以達成「低成本、高可用、免維護」的目標。

## 方案重點
1.  **後端轉型**：從 Flask (WSGI) 轉向 Cloudflare 異步 Worker 架構。
2.  **資料庫遷移**：從本地 SQLite 檔案轉向 D1 雲端資料庫。
3.  **安全性強化**：利用 Cloudflare Access 或 自定義 Worker Auth 取代本地 Session。

## 驗證計畫

### 自動化測試
*   使用 `wrangler dev` 在本地模擬 D1 環境進行 API 整合測試。
*   撰寫針對 `worker.py` 的單元測試，模擬 Request 物件並驗證 Response 狀態碼。

### 手動驗證流程
1.  **D1 初始化測試**：執行 `wrangler d1 execute` 建立表結構並檢查是否正確。
2.  **看板同步測試**：開啟兩個瀏覽器分頁（管理端與顯示端），更改內容後驗證顯示端是否在 5 秒內更新。
3.  **離線響應測試**：切換網路狀態，驗證看板前端是否能正確處理 502/504 錯誤而非直接白屏。
