# POR 系統項目修改功能實作計畫

本計畫旨在為「進度管理」增加項目內容修改功能，僅限具備新增權限的使用者操作。

## 擬議變更

### 1. 後台邏輯 (Python / Flask)

#### [MODIFY] [main.py](file:///d:/PythonWorking/POR/PORsystem/main.py)
- 新增 `@app.route('/edit_item/<int:item_id>', methods=['GET', 'POST'])` 路由。
- **GET 模式**：讀取指定項目並渲染 `settings.html`，同時傳入項目物件以便填入表單。
- **POST 模式**：更新資料庫中的項目內容與備註，並觸發 `touch_system_update()`。
- **權限檢查**：確保只有管理員或具備 `can_add_order` 權限的用戶可進入。

### 2. 前端表單 (HTML / Jinja2)

#### [MODIFY] [settings.html](file:///d:/PythonWorking/POR/PORsystem/templates/settings.html)
- 判斷是否存在 `item` 物件。
- 若為編輯模式：
    - 將標題改為「修改看板項目」。
    - 將表單 Action 改為 `/edit_item/{{ item.id }}`。
    - 在文字方塊中預填 `item.content` 與 `item.remark`。
    - 將按鈕文字改為「確認修改」。

### 3. 下一步引導 (HTML)

#### [MODIFY] [manage.html](file:///d:/PythonWorking/POR/PORsystem/templates/manage.html)
- 在「內容」欄位或對應位置加入「修改」按鈕。
- 僅在用戶具備 `can_add_order` 權限或為管理員時顯示。

## Docker 映像檔導出與隔離發佈方案

為了實現「快速部署」且「保護原始碼」，我們採用以下流程：

### 1. 建立建檔排除機制
- 已建立 `.dockerignore`，確保映像檔內不包含 `doc/`、`.git`、`.gitignore` 以及 `instance/*.sqlite` 等敏感文檔與數據。

### 2. 映像檔封裝 (此步驟在您的開發主機執行)
1. 執行 `docker-compose build` 建立最新的映像檔。
2. 執行 `docker save -o POR_Signboard_v1.tar signboard_app:latest` 將映像檔導出為單一檔案。

### 3. 隔離發佈 (交付給客戶/現場)
- 交付物：
    - `POR_Signboard_v1.tar` (核心映像檔)
    - `docker-compose.release.yml` (不含 build 指令，僅引用映像檔)
    - `🚀點我啟動看板.bat` (修改版，使用 `docker load` 載入映像檔)
- **優點**：現場電腦不需要原始程式碼 (`main.py` 等) 即可運行，且啟動快速，不需額外下載依賴。

## 驗證計畫

### 手動驗證
1. 在本地執行導出指令。
2. 嘗試在另一台乾淨的環境（或模擬環境）載入 `.tar` 檔。
3. 確認無需 `main.py` 即可透過 Docker 啟動看板。
