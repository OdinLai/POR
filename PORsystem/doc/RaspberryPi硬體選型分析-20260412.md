# Raspberry Pi 硬體負載與穩定性分析報告 (POR 雲端版)

## 1. 穩定性優先策略 (針對國際市場)
若未來涉及國外客戶部署，穩定性將是第一優先考量：
*   **硬體選擇**：強烈建議使用 **Raspberry Pi 3B+** 或 **Raspberry Pi 4B**。
    *   **理由**：雖然 Zero 2 W 體積小，但在熱管理與記憶體餘裕上較弱。3B+/4B 的擴展性與穩定性在工業/商用環境已有大量實證。
*   **定時重啟**：實施凌晨 4 點自動重啟，是確保 WebSignage (網頁看板) 永不當機的低成本、高效益手段。

## 2. 規格建議表

| 情境 | 推薦型號 | 記憶體配置 | 重點優勢 |
| :--- | :--- | :--- | :--- |
| **商用穩定型** | **Raspberry Pi 3B+** | 1GB | 散熱優中產能穩、與 POR 網頁渲染負載完美契合。 |
| **高效能展示型** | **Raspberry Pi 4B** | 2GB+ | 支援雙 4K 輸出、CPU 運算極快，適合複雜動畫。 |

## 3. 部署優化建議
*   **網路穩定性**：國外客戶部署時，搭配 **Cloudflare Pages/Workers** 能將內容緩存在全球邊緣節點，顯著降低跨境訪問的延遲，確保看板即時同步。
*   **記憶體監控**：限制 Chrome 記憶體於 512MB 可防止瀏覽器因長效運行而耗盡系統資源。
