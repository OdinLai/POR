# 投影看板卷頁邏輯優化計畫 - 2026-04-18

目前的卷頁邏輯使用硬編碼的 100 像素作為行高進行滾動，當內容換行或顯示比例變動時會導致對不準。本計畫將改為基於 DOM 元素的精確定位捲動。

## 擬議變更

### 網頁前端 (show.html)

#### [MODIFY] [show.html](file:///d:/PythonWorking/POR/PORsystem/templates/show.html)
- 修改 `startAutoScroll` 函式：
  - **啟動判斷**：比較 `wrapper.clientHeight` 與 `container.scrollHeight`，溢出才啟動。
  - **步進邏輯**：使用 `setInterval` 每 3 秒執行一次捲動。
  - **位置計算**：每次捲動到 `rows[currentIndex].offsetTop`。
  - **終點處理**：當捲動到最後一行位於畫面頂端時（或到達最大捲動高度時），額外停留 3 秒，然後返回 `currentIndex = 0` 重新開始循環。
  - **視覺空間**：末端固定補足 25 筆影子空行，確保捲動空間。

## 驗證計畫

### 手動驗證

   - 測試資料量多時，確認最後一列能完全置頂。
