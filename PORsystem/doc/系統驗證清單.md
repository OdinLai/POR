# 投影看板系統 (POR) 系統驗證測試清單

這份清單旨在協助您在正式部署前，確認系統各項功能是否運作正常。

## 1. 🪟 啟動與顯示驗證 (Kiosk Mode)
- [x] **獨立視窗**：雙擊 `🚀點我啟動看板.bat` 啟動後，是否彈出一個**沒有邊框、沒有網址列**的 Chrome 視窗？
- [x] **不干涉測試**：原本開啟的 Chrome 視窗（例如您的工作頁面）是否依然正常，沒有受看板啟動影響？
- [x] **視覺呈現**：看板頁面是否有「淺藍色漸層表頭」且表格內容為「灰白相間」的條紋樣式？
- [x] **預設標題**：未修改前，首頁標題是否顯示「中國到貨看板系統」？

## 2. 🏷️ 看板標題管理
- [x] **標題修改**：進入 `http://localhost:5000/manage`，在最上方的「系統設定」輸入新標題（例如：中東到貨看板），並點擊「更新標題」。
- [ ] **同步檢查**：切換回看板頁面（`show` 頁面），標題是否已即時更新？

## 3. 📝 訂單與流程轉移
- [x] **建立訂單**：在 `http://localhost:5000/settings` 建立一筆測試資料（隨意輸入內容與備註）。
- [ ] **看板呈現**：看板的「訂貨」欄位是否出現該筆資料，且排版整齊？
- [ ] **流程轉移**：在「管理進度」頁面，點擊「轉移至到貨」。
- [ ] **位置確認**：看板上的資料是否從「訂貨」區塊移動到了「到貨」區塊？
- [ ] **備註顯示**：在看板頁面，將滑鼠移到該項目上，是否會懸浮顯示完整的備註內容？

## 4. 🔄 自動刷新與效能
- [ ] **30秒自動刷新**：在看板頁面不動，等待 30-40 秒，頁面是否會自動重整以同步最新數據？

## 5. 🔑 權限與安全
- [ ] **非管理員權限**：嘗試建立一個一般員工帳號，並僅給予「新增訂貨」權限，測試其是否無法執行「清空」或「轉移」動作。
