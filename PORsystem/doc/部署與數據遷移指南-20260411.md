# 投影看板系統：部署與數據遷移指南 (2026-04-11)

本指南旨在說明如何將開發完成的 POR 系統部署至客戶現場，並確保數據的持久性與安全性。

## 1. 核心觀念：資料持久化
Docker 容器本身是「臨時的」，程式運行產生的所有資料都應儲存在 **Volume (掛載目錄)** 中。在本專案中，所有的資料庫與用戶設定都存在：
`./instance/signboard.db`

---

## 2. 推薦部署流程 (隨身碟搬移法)
這是最省心、且適合在客戶現場（可能有網路限制）的操作方式：

### A. 在您的開發機 (Windows)
1. 確保代碼已完成所有測試。
2. 執行 Git Push 確保代碼備份。
3. **打包專案**：將整個專案資料夾（排除 `__pycache__`）壓縮成 `POR-V1.zip`。

### B. 在客戶現場機器 (Ubuntu/樹莓派)
1. 安裝 Docker 與 Docker Compose。
2. 解壓縮專案 zip 檔至 `/home/ubuntu/POR`。
3. **啟動系統**：
   ```bash
   cd /home/ubuntu/POR
   docker-compose up -d --build
   ```
4. **驗證**：打開瀏覽器訪問 `http://localhost:5000`。

---

## 3. 如何搬移現有的資料 (數據遷移)
如果您在測試環境已經產生了重要數據，想直接帶到現場：
1. **停止容器**：`docker-compose down`
2. **複製資料檔**：將 `./instance/signboard.db` 複製出來。
3. **覆蓋至現場**：將檔案放入現場機器的 `./instance/` 同目錄下。
4. **重新啟動**：系統會自動抓取舊的資料庫。

---

## 4. 故障排除：防斷聯策略 (Tailscale)
為了避免現場主機變更 IP 導致維護困難，強烈建議：
1. 在現場主機安裝 [Tailscale](https://tailscale.com/)。
2. 啟動後，您可以透過 Tailscale 分發的固定靜態 IP (100.x.x.x) 從公司連入管理，完全無視現場路由器的複雜設定。
