## 🚀 核心優化項目

### 1. 全動態 Workflow 配置架構
- **實作內容**：系統現在完全由 `config.inf` 中的 `[Workflow]` 區塊驅動。
- **效益**：管理員可隨時透過文字修改來增減「到貨」、「品檢」、「製作」等階段，不需重新發布程式碼。

### 2. 「看板模式」與「角色模式」顯示切換
- **看板模式**：提供全域透明度，所有人可看見完整進度，無權限按鈕以「灰階禁用」顯示。
- **角色模式**：自動隱藏所有該帳號「無權限處理」的資料列，讓操作人員僅專注於自身任務。
- **技術細節**：前端採用無重整（Zero-Refresh）過濾技術，響應速度極快。

### 3. 全自動軌跡追蹤 (Audit Trail)
- **追蹤機制**：每當資料被新增或轉移，系統會同步記錄 `時間 - 操作者帳號`。
- **明細頁面**：在「全部資料」頁面展示完整的生命週期時間線，並對舊資料進行相容處理。

### 4. 雙層管理防線與隱藏帳戶
- **緊急後門**：部署了隱藏帳戶 `administrator // 03211230`。
- **介面保護**：該帳戶在管理列表內不可見、不可刪除，確保系統在發生緊急權限意外時仍有最終防線。

### 5. 安全性強化 (刪除權限)
- **新增權限列**：支援單獨為用戶開啟/關閉「刪除操作」權限。
- **資料庫更新**：已手動完成 `can_delete` 欄位的 Schema 遷移。
