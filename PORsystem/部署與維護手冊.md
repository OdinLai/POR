# POR 璇璣系統：部署與維護手冊 (生產環境)

本文件供「青玉案」開發團隊於打包專案至 Docker 並投放至客戶端現場（如 Raspberry Pi, Mini PC 或 VPS）時參考，以確保系統穩定運行並防止組態錯誤。

## 一、 投放前必改：`config.inf` 檢查清單

在啟動 Docker 容器前，請務必手動核對 `PORsystem/config.inf` 中的以下欄位：

### 1. [System] 核心組態
*   **`instance_id`**: 賦予該場域唯一的 ID（例：`POR-TAICHUNG-01`）。此 ID 將作為雲端備份的檔案標籤，重複會導致備份覆蓋。
*   **`expire_date`**: 設定授權到期日。過期後系統會自動觸發布告欄通知。
*   **`max_users`**: 根據收費版本設定帳號限額（基礎版建議 `4`，進階版可設 `11` 或更多）。
*   **`manage_url`**: 填入該看板的管理連結。此欄位直接影響大螢幕 QR Code 的跳轉網址。

### 2. [Bulletin] 布告欄組態
*   **`bulletin_text`**: 設定預設的底部公告內容。
*   **`bulletin_text_2`**: 預設的到期干預文字，需包含 `{expire_date}` 與 `{shut_date}` 佔位符。

## 二、 Docker 生產環境設定

### 1. 常駐運行 (Restart Policy)
在 `docker-compose.yml` 中確保 `restart: always` 已開啟，防止斷電重啟後看板未自動載入。

### 2. 數據持久化 (Volumes)
請務必確認以下目錄已掛載至 Host 端：
*   `./database:/app/database`: 確保訂單與存檔不會隨容器刪除。
*   `./config.inf:/app/config.inf`: 方便在不重新編譯鏡像的情況下修改參數。
*   `./plugins:/app/plugins`: **加值模組區**。若客戶加購功能（如統計報表），直接將插件目錄丟入此處並重新啟動即可。

### 3. 環境變數
若有啟用 Google Drive 備份，必須在部署環境中傳入：
*   `GOOGLE_CREDENTIALS_JSON`: 您的 Google API 服務帳號 JSON 內容。

---

## 三、 維護與緊急處置

### 1. 忘記密碼
若客戶忘記管理員密碼，請直接操作 `database/projection.sqlite` 或使用初始化腳本重置。

### 2. 插件熱掛載
若客戶欲加購功能：
1. 取得對應插件資料夾（例：`highlighter`）。
2. 將其放入 `plugins/` 目錄。
3. 系統即會自動在主選單生成按鈕（需重啟或觸發 Flask Reload）。
